Get ReasonDB running and execute your first AI-powered search. No complex setup, no configuration headaches—just intelligent document retrieval.

## What You'll Build

By the end of this guide, you'll have:
- A running ReasonDB instance
- An ingested document transformed into a navigable knowledge tree
- Your first LLM-guided search with extracted answers

## Prerequisites

## Installation

**For users:** Use Docker (or the published image). No repository clone or build required.

<Tabs>
  <Tab title="Docker (recommended for users)">
    <Steps>
      <Step title="Create a project folder and env file">
        No need to clone the repo. Create a folder and add your LLM config:

        ```bash
        mkdir reasondb-app && cd reasondb-app
        cat > .env <<'EOF'
        REASONDB_LLM_PROVIDER=anthropic
        REASONDB_LLM_API_KEY=sk-ant-xxxxx
        EOF
        ```

        <Accordion title="Supported LLM Providers">
          | Provider | `REASONDB_LLM_PROVIDER` | Best For |
          |----------|------------------------|----------|
          | **Anthropic Claude** | `anthropic` | Best reasoning quality |
          | **OpenAI GPT-4** | `openai` | Widely available |
          | **Google Gemini** | `gemini` | Cost-effective |
          | **Google Vertex AI** | `vertex` | GCP, enterprise Gemini |
          | **AWS Bedrock** | `bedrock` | AWS, Claude and others |
          | **Cohere** | `cohere` | Enterprise features |
          | **Ollama (local)** | `ollama` | Privacy, no API key |
        </Accordion>
      </Step>

      <Step title="Add a minimal docker-compose.yml">
        Save this as <code>docker-compose.yml</code> in the same folder:

        ```yaml
        services:
          reasondb:
            image: ajainvivek/reasondb:latest
            ports:
              - "4444:4444"
            volumes:
              - reasondb_data:/data
            env_file:
              - .env
            environment:
              REASONDB_HOST: "0.0.0.0"
              REASONDB_PATH: "/data/reasondb.redb"
            restart: unless-stopped

        volumes:
          reasondb_data:
        ```
      </Step>

      <Step title="Start ReasonDB">
        ```bash
        docker compose up -d
        ```

        The image is pulled once; no build step. Data persists in a Docker volume.
      </Step>

      <Step title="Verify">
        ```bash
        curl http://localhost:4444/health
        ```

        API and Swagger UI: **http://localhost:4444/swagger-ui/**
      </Step>
    </Steps>

    <Tip>
    Optional: install the CLI with <code>brew tap reasondb/tap && brew install reasondb</code> to run <code>reasondb search</code> and <code>reasondb query</code> against your server.
    </Tip>
  </Tab>

  <Tab title="Docker run (one-liner)">
    If you prefer a single command without compose:

    ```bash
    docker run -d --name reasondb -p 4444:4444 \
      -e REASONDB_LLM_PROVIDER=anthropic \
      -e REASONDB_LLM_API_KEY=sk-ant-xxxxx \
      -v reasondb_data:/data \
      -e REASONDB_PATH=/data/reasondb.redb \
      ajainvivek/reasondb:latest
    ```

    Replace the env values with your provider and API key. Data is stored in the <code>reasondb_data</code> volume.
  </Tab>

  <Tab title="Homebrew (CLI + local server)">
    For macOS/Linux when you want the CLI and a local server (no Docker):

        ```bash
        brew tap reasondb/tap
        brew install reasondb
        reasondb config init   # set LLM provider and API key
        reasondb serve
        ```

    Server at **http://localhost:4444**. Config in <code>~/.config/reasondb/config.toml</code>.
  </Tab>

  <Tab title="For contributors (build from source)">
    Clone the repo and build with Rust. Only needed if you're developing or contributing.

    <Steps>
      <Step title="Clone and Build">
        ```bash
        git clone https://github.com/reasondb/reasondb.git
        cd reasondb
        cargo build --release
        ```

        <Tip>
        First build takes 2-3 minutes. Subsequent builds are fast.
        </Tip>
      </Step>

      <Step title="Configure Your LLM">
        Run the interactive setup wizard:

        ```bash
        ./target/release/reasondb config init
        ```

        Or configure directly:

        ```bash
        reasondb config set llm.provider anthropic
        reasondb config set llm.api_key sk-ant-xxxxx
        ```

        <Note>
        Your API key is stored securely in `~/.config/reasondb/config.toml`.
        </Note>
      </Step>

      <Step title="Start the Server">
        ```bash
        reasondb serve
        ```

        You'll see:
        ```
          ____                            ____  ____  
         |  _ \ ___  __ _ ___  ___  _ __ |  _ \| __ ) 
         | |_) / _ \/ _` / __|/ _ \| '_ \| | | |  _ \ 
         |  _ <  __/ (_| \__ \ (_) | | | | |_| | |_) |
         |_| \_\___|\__,_|___/\___/|_| |_|____/|____/ 

        AI-Native Document Intelligence

          Database: data/reasondb.redb
          API: http://127.0.0.1:4444
          Swagger UI: http://127.0.0.1:4444/swagger-ui/
          LLM Provider: anthropic
        ```

        <Info>
        Visit the Swagger UI to explore the full API interactively.
        </Info>
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Your First Intelligent Search

Now let's see ReasonDB in action. We'll ingest a document and ask it questions.

### Step 1: Create a Table

Tables organize your documents into searchable collections:

```bash
curl -X POST http://localhost:4444/v1/tables \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Knowledge Base",
    "description": "Technical documentation and guides"
  }'
```

```json
{
  "id": "tbl_abc123",
  "name": "Knowledge Base",
  "document_count": 0
}
```

<Warning>
Save the `id`—you'll need it for ingestion.
</Warning>

### Step 2: Ingest a Document

Let's ingest a structured document. ReasonDB will automatically:
1. Parse the Markdown structure
2. Build a hierarchical tree from headings
3. Generate LLM summaries for intelligent navigation

```bash
curl -X POST http://localhost:4444/v1/ingest/text \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Machine Learning Guide",
    "table_id": "tbl_abc123",
    "content": "# Machine Learning Guide\n\nA comprehensive introduction to machine learning concepts and techniques.\n\n## Supervised Learning\n\nSupervised learning uses labeled data to train models. The algorithm learns to map inputs to known outputs.\n\n### Classification\n\nClassification predicts discrete categories. Examples include spam detection, image recognition, and medical diagnosis. Common algorithms: logistic regression, decision trees, random forests, and neural networks.\n\n### Regression\n\nRegression predicts continuous values. Examples include price prediction, demand forecasting, and risk assessment. Common algorithms: linear regression, polynomial regression, and gradient boosting.\n\n## Unsupervised Learning\n\nUnsupervised learning finds patterns in unlabeled data without predefined outputs.\n\n### Clustering\n\nClustering groups similar data points together. Applications include customer segmentation, anomaly detection, and document organization. Popular algorithms: K-means, DBSCAN, and hierarchical clustering.\n\n### Dimensionality Reduction\n\nDimensionality reduction compresses data while preserving important information. Used for visualization, noise reduction, and feature extraction. Common techniques: PCA, t-SNE, and autoencoders."
  }'
```

Text ingestion is **async** — it returns immediately with a job ID (HTTP 202):

```json
{
  "job_id": "job_abc123",
  "status": "queued",
  "created_at": "2026-01-30T10:00:00Z",
  "updated_at": "2026-01-30T10:00:00Z"
}
```

Poll for completion:

```bash
curl http://localhost:4444/v1/jobs/job_abc123
```

When complete, the job result contains:

```json
{
  "job_id": "job_abc123",
  "status": "completed",
  "result": {
    "document_id": "doc_xyz789",
    "title": "Machine Learning Guide",
    "total_nodes": 7,
    "max_depth": 3,
    "stats": {
      "chars_extracted": 1250,
      "chunks_created": 6,
      "nodes_created": 7,
      "summaries_generated": 7,
      "total_time_ms": 4200
    }
  }
}
```

**What just happened?** ReasonDB transformed your flat text into a navigable knowledge tree:

<Mermaid>
{`graph TD
    A[📚 Machine Learning Guide] --> B[Supervised Learning]
    A --> C[Unsupervised Learning]
    B --> D[Classification]
    B --> E[Regression]
    C --> F[Clustering]
    C --> G[Dimensionality Reduction]`}
</Mermaid>

Each node has an LLM-generated summary enabling intelligent navigation.

### Step 3: Ask Questions

Now the magic happens. Use RQL to ask a natural language question:

```sql
SELECT * FROM knowledge_base
REASON 'What algorithms are used for classification?'
```

```json
{
  "documents": [{
    "id": "doc_xyz789",
    "title": "Machine Learning Guide",
    "table_id": "tbl_abc123",
    "tags": [],
    "metadata": {},
    "total_nodes": 7,
    "score": 0.94,
    "highlights": ["Classification predicts discrete categories..."],
    "answer": "Common classification algorithms include logistic regression, decision trees, random forests, and neural networks. These are used for tasks like spam detection, image recognition, and medical diagnosis.",
    "confidence": 0.94
  }],
  "total_count": 1,
  "execution_time_ms": 1650
}
```

**Notice the difference from vector search:**
- ReasonDB navigated the document tree to find the exact relevant section
- You get a synthesized `answer` with a `confidence` score
- `highlights` show the matched content within the document

### Try More Queries

```sql
-- Cross-section question
SELECT * FROM knowledge_base
REASON 'What are the differences between supervised and unsupervised learning?'

-- Keyword search (BM25, fast)
SELECT * FROM knowledge_base SEARCH 'K-means clustering' LIMIT 5

-- Combine filters with reasoning
SELECT * FROM knowledge_base
WHERE tags CONTAINS ANY ('unsupervised')
REASON 'How does dimensionality reduction help with visualization?'
```

## Using the CLI

ReasonDB includes a powerful command-line interface:

```bash
# Natural language search
reasondb search "What is the difference between classification and regression?"

# Interactive RQL REPL
reasondb query

# Execute RQL directly
reasondb query -q "SELECT * FROM 'Knowledge Base' SEARCH 'clustering' LIMIT 5"

# Manage tables
reasondb tables list
reasondb tables get tbl_abc123

# Health check
reasondb health
```

## View Document Structure

Inspect how your document was transformed:

```bash
curl http://localhost:4444/v1/documents/doc_xyz789/tree
```

This returns the complete tree with all nodes, summaries, and relationships.
